ExpressJS

A Modern* Server-Side Solution For Form Handling & More

ExpressJS is a library which works with the same language used on the frontend - JavaScript... only on the server side. By using NodeJS or the modern alternatives (Deno/Bun), one can easily create an HTTP server using Express which can then handle individualistic routes which the frontend can request for data to be returned or for a specific action to be done. ExpressJS offers a paradigm called Middleware, which allows developers to specify a function (callback) for which every request is piped through, allowing for generalized parsing of form data sent in a POST/PUT/etc... request. This is quite useful then because arbitrary code on the backend can then be run to do something with the form data, such as saving it to a database and/or using it in a more complicated system. This is much more than what is possible on the frontend, which has CORS limitations and can't be used for secure database writes. ExpressJS is a very popular library, with over 5 million weekly downloads on NPM, I've used it myself to build my own account system that uses a HTML forms (and the HTMX library) to create/login/reset password/etc... and it or a similar solution would be what I'd choose again. Although it's great, it's not perfect, and other solutions exist, such as frontend-based form handling with AJAX or using a more traditional server-side language like PHP or ASP.NET.
